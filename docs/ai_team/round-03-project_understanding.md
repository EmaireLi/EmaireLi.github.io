# Round 3 Project Understanding

Date: 2026-07-08

## Current Website Positioning

This is still a static GitHub Pages personal archive, but after the first two AI-team rounds it now has a stronger portfolio layer:

- Homepage project cards for SmartLabeling and Yomii.
- Dedicated static case-study pages under `projects/`.
- A homepage `Proof Map` with hiring snapshot, capability evidence, ownership notes, and review routes.
- AI-team planning/evaluation/decision records under `docs/ai_team/`.
- Existing blog/archive/search/editor/XHS import/guestbook surfaces remain intact.

## Current Advantages

- The homepage now answers "what can this person build?" faster than before.
- Project evidence is linked from multiple places: proof matrix, project cards, and case pages.
- The repo keeps a visible decision trail and verification routine.
- The implementation stays compatible with plain HTML/CSS/JavaScript and GitHub Pages.

## Current Defects

- The strongest technical project, SmartLabeling, is still mostly text proof; it lacks visual workflow evidence.
- The site-as-project story is visible only as one matrix row, not as a self-contained engineering artifact.
- Blog/post reading experience has not been improved beyond existing archive/search behavior.
- Existing local search is useful but basic; it does not expose ranking, snippets, or match reasons.
- Project review routes still link at a high level; there is no cross-project code tour index.

## Highest-Value Improvement Directions

1. Add truthful visual proof for SmartLabeling, ideally a static or lightly interactive mask/workflow strip tied to the existing case study.
2. Build a site-as-project engineering colophon if it can stay concise and evidence-backed.
3. Improve content discovery with explainable local search or curated reading paths, but avoid risky changes to `script.js` unless the value is clear.
4. Improve long-post reading UX only after recruiter/technical proof surfaces are stronger.

## Technical Stack

- Static HTML/CSS/JavaScript.
- No package manager or `package.json`.
- Blog manifest generated by `scripts/generate-posts-manifest.js`.
- XHS import via `scripts/import-xhs-notes.js`.
- Guestbook Worker/D1 code under `guestbook/`.
- Visual style centralized in `styles.css`.
