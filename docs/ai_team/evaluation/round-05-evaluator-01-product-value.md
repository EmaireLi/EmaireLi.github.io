## Recommended Features

Primary recommendation: Site Pipeline Lens.

It best satisfies the corrected scope because it turns the personal website itself into the proof object: editor, XHS import, generated manifest, archive, search, guestbook, and GitHub Pages constraints. It has strong technical showcase value without requiring new infrastructure.

Best bundle: Site Pipeline Lens + Curated Archive Lanes + Personal Positioning Microcopy Pass.

That combination improves trust, content discovery, visual hierarchy, and static-site engineering presentation while staying compatible with plain HTML/CSS/JS.

## Eliminated Features

- Project Constraint Cards: useful for case studies, but pulls attention back toward external GitHub projects instead of improving the personal site itself.
- Role Readiness Scorecard: recruiter value exists, but the self-rating pattern is risky and less credible than proof-based routing.
- Decision Receipts Strip: defer unless folded into the Site Pipeline Lens. As a standalone feature, it can feel too internal.
