## Recommended Features

1. **First-Viewport Identity / Positioning Panel** - strongest first-visit comprehension fix.
2. **Start Here Visitor Routes** - static links solve most route needs more accessibly than filters.
3. **Proof Narrative Ladder / Proof-to-Project Bridge** - turns existing sections into a guided reading path.
4. **Project Card Scan Upgrade** - improves recruiter scanning and technical reviewer orientation.
5. **Accessibility Comfort Pass** - should be considered during implementation, especially focus states, touch targets, contrast, and reduced motion.

## Deferred / Eliminated

- Mobile section jump bar: useful after top-page routing is clearer.
- Curated archive, search reasons, reading TOC, and contact prompts: good follow-ups but not the current bottleneck.
- WebGL/canvas and visual archive image rail: too much accessibility and layout risk for this round.
- Capability filters/reviewer mode: avoid as primary navigation because they add state complexity.
