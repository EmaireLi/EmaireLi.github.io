# Round 05 Evaluation - Evaluator 02 Technical Differentiation

Scores are weighted: value 40%, technical showcase 25%, implementation cost/risk 20%, long-term benefit 15%.

| Rank | Feature Family | Score | Reason |
|---:|---|---:|---|
| 1 | Search Match Reasons/Context | 86 | Strongly improves content discovery and shows real static-site engineering: indexing, excerpts, tags, match explanations, and relevance cues. |
| 2 | Curated Archive Lanes/Windows | 84 | High user value with low risk. Makes the archive feel intentional instead of chronological only. |
| 3 | Evidence Freshness Markers | 81 | Improves trust by showing when claims, links, projects, or posts were last checked. |
| 4 | Reading Route Cards/Featured Reading Path | 79 | Excellent for guiding visitors through the site. Strong conversion and discovery value. |
| 5 | Personal Positioning Microcopy Pass | 77 | Very high value-to-cost ratio. Low technical showcase, but immediately improves hierarchy and conversion. |
| 6 | Site Pipeline Lens | 75 | Best pure technical differentiation candidate. It can show how the static site imports, generates manifests, and publishes content. |
| 7 | Contact Intent Panel | 72 | Strong conversion with low implementation cost. Less technically distinctive. |
| 8 | Visual Archive Preview Strip | 70 | Good visual hierarchy and browsing affordance. Risk is decorative execution. |
| 9 | Decision Receipts Strip | 68 | Good trust/engineering signal if it explains site-level decisions. |
| 10 | Homepage Review Minimap | 64 | Interesting interaction, but uncertain value unless the homepage is long. |
| 11 | Account Identity Map | 62 | Useful for trust and identity verification, but technically light. |
| 12 | Project Constraint Cards | 59 | Only worthwhile if framed around this website's constraints. |
| 13 | Role Readiness Scorecard | 56 | Risks feeling self-scored or resume-gimmicky. |
| 14 | AI Boundary Ledger | 53 | Trust-oriented, but likely too niche unless AI-generated/imported content is central. |

## Recommended Features

1. Search Match Reasons/Context
2. Curated Archive Lanes/Windows
3. Evidence Freshness Markers
4. Personal Positioning Microcopy Pass

## Conditional Feature

Site Pipeline Lens is worth doing if it is presented as part of the site's credibility: how this site is maintained, how posts are imported, why assets are local, and what is generated automatically. It should not become a developer dashboard that distracts from the personal website.

## Eliminated Features

- AI Boundary Ledger: eliminate for now unless the site prominently uses AI-assisted content and visitors need disclosure.
- Role Readiness Scorecard: eliminate because it risks weakening credibility through self-rating.
- Project Constraint Cards: eliminate unless narrowed to constraints of the personal website itself.
- Homepage Review Minimap: eliminate for this round unless the homepage is long enough to justify a navigation/minimap layer.
