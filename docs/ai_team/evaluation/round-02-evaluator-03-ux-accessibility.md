# Round 2 Evaluation - Evaluator 3: UX and Accessibility Critic

## Ranking

| Rank | Feature | Score | UX/accessibility reason |
|---:|---|---:|---|
| 1 | Hiring Snapshot | 86 | Best scan-speed improvement. Gives visitors role, proof, and next action fast with low mobile/accessibility risk. |
| 2 | Capability Evidence Matrix | 83 | Turns claims into verifiable proof. Strong comprehension value if kept short and link-based. |
| 3 | Recruiter / Proof-Based Reading Paths | 77 | Reduces decision fatigue for different visitor types. Good mobile fit as simple cards or lists. |
| 4 | Site-as-Project / Colophon | 73 | Makes hidden engineering work visible with low implementation risk. Maintenance burden is moderate because docs can stale. |
| 5 | Guided Code Review Route / Code Tour Index | 72 | Very useful for technical reviewers, low cost, but external repo links may drift. |
| 6 | Static Resume / One-Page Profile | 71 | Practical recruiter artifact. Risk is duplication and staleness. |
| 7 | Project Visual Proof Gallery / Proof Strip | 69 | Improves comprehension quickly, but only if real screenshots/assets exist. Placeholder visuals would hurt trust. |
| 8 | Article TOC + Reading Progress | 66 | Good reader UX, but sticky/mobile behavior must be careful and it touches shared JS. |
| 9 | Explainable Local Search | 64 | Useful and technically visible, but higher JS complexity, Chinese tokenization limits, and more regression risk. |
| 10 | Bilingual Visitor Summary | 63 | Strong comprehension benefit for non-Chinese recruiters, but translation maintenance is real. |
| 11 | Engineering Decision Log | 61 | Shows judgment, but can slow scanning if verbose. Better as short linked cards. |
| 12 | Learning Trajectory / Topic Timeline | 58 | Helpful narrative, but weaker if content density is uneven. |
| 13 | Now / Build Log | 55 | Signals freshness, but high stale-content risk. |
| 14 | Recruiter FAQ / Contact Intent Guide | 52 | Low cost and accessible, but low technical signal. |
| 15 | Command Palette | 49 | Nice craft signal, but poor mobile value and non-trivial accessibility burden. |
| 16 | SmartLabeling Mask Visualizer / Mini Vision Lab | 47 | Strong display value, but costly, harder to make accessible, and must avoid misleading simulated AI. |
| 17 | WebGPU / Shader / GPU Workbench / Archive Constellation | 35 | High novelty, low recruiter comprehension, heavy QA/accessibility burden, weak maintenance fit for this static site. |

## Recommended

1. Hiring Snapshot
2. Capability Evidence Matrix
3. Recruiter / Proof-Based Reading Paths

Second wave:

1. Site-as-Project / Colophon
2. Guided Code Review Route
3. Project Visual Proof Gallery, only with accurate local assets

## Eliminated For Round 2

- WebGPU Graphics Lab, Shader Notes, GPU Image Workbench, Archive Constellation
- Command Palette
- Mini Vision Lab / SmartLabeling Mask Visualizer
- Auto Tags / Metadata Assistant / Semantic-style Search claims
