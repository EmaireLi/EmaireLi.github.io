# Round 4 Brainstorm - Agent 4: AI Applications Expert

Date: 2026-07-09

## Scope

Only personal website visual design, homepage structure, interaction, and content presentation. No paid APIs, external accounts, or external repository work.

## Approach Options

1. **AI-proof presentation layer:** recommended; static and evidence-linked.
2. **AI-inspired interactions:** local-data chips and maps without real AI.
3. **Generated visual decoration:** lower priority.

## Feature Ideas

### 1. AI Workbench Snapshot

- **User value:** Clarifies what "AI tooling" means in the portfolio.
- **Why it fits:** Proof Map already has AI-related evidence but could be easier to scan.
- **Technical approach:** Add rows for project, AI role, human role, proof link, limitation.
- **Cost:** Low.
- **Display effect:** Factual AI applications panel.
- **Risk:** Broad wording can overclaim.

### 2. Runtime Honesty Badges

- **User value:** Shows what is local, mocked, optional, or static.
- **Why it fits:** The site values verification and avoids fake proof.
- **Technical approach:** Add badges and inline notes to project/proof cards.
- **Cost:** Low.
- **Display effect:** Lightweight trust signal.
- **Risk:** Badges need maintenance.

### 3. Reviewer Intent Switcher

- **User value:** Visitors can quickly see evidence that matters to them.
- **Why it fits:** Proof Map already has visitor routes.
- **Technical approach:** Use plain JS buttons to highlight recruiter, technical, AI tooling, or process evidence.
- **Cost:** Medium.
- **Display effect:** Personalized static homepage.
- **Risk:** Hidden content can reduce discovery; prefer highlighting.

### 4. Local Proof Query Chips

- **User value:** One-click exploration by theme.
- **Why it fits:** The site already has local search and post manifest.
- **Technical approach:** Chips fill/filter existing search with terms like `AI tooling`, `full-stack`, `engine learning`, `writing`.
- **Cost:** Low to medium.
- **Display effect:** Search feels guided and intelligent without real AI.
- **Risk:** Weak results if tags/keywords are inconsistent.

### 5. Learning Trajectory Timeline

- **User value:** Shows movement from Java/full-stack toward engine development and AI tooling.
- **Why it fits:** Homepage copy already frames a long-term archive.
- **Technical approach:** Static timeline using posts, projects, and site milestones.
- **Cost:** Medium.
- **Display effect:** Coherent growth story.
- **Risk:** Timeline can become noisy.

### 6. Project Decision Receipts

- **User value:** Shows product and engineering judgment.
- **Why it fits:** AI-team docs already exist.
- **Technical approach:** Homepage receipt cards: decision, why, evidence, tradeoff.
- **Cost:** Low.
- **Display effect:** Makes process credible.
- **Risk:** Internal process language can distract.

### 7. Static Content Relationship Map

- **User value:** Helps visitors discover connections between projects, posts, and themes.
- **Why it fits:** The site is an archive.
- **Technical approach:** Hand-authored JSON or data attributes render simple nodes/links in HTML/CSS.
- **Cost:** Medium.
- **Display effect:** Knowledge-graph feel without external services.
- **Risk:** Can look gimmicky.

## Recommendation

Start with Runtime Honesty Badges, Reviewer Intent Switcher, and Local Proof Query Chips if AI presentation is the goal. Keep the implementation honest and static.
