# Round 05 Brainstorm - Agent 05 Recruiting Manager

Scope: improve the personal website as a hiring signal. The best feature should make a recruiter or technical interviewer understand capability faster.

## Direction Options

- Proof depth layer: show how the site itself is engineered and verified.
- Recruiter conversion layer: improve contact and interview entry points.
- Archive credibility layer: make writing easier to assess.

Recommended direction: proof depth layer first.

## Feature Ideas

### 1. Site Pipeline Evidence Lens

- User value: a recruiter sees that the site is an actively maintained system.
- Fit: the site has editor, import, manifest generation, search, archive, guestbook, and GitHub Pages.
- Technical plan: homepage pipeline from Write to Publish with proof links.
- Estimated cost: low to medium.
- Showcase effect: strong practical engineering signal.
- Risk: may be too technical for some visitors unless the copy stays clear.

### 2. Decision Receipts Strip

- User value: reviewers see that improvements are chosen and verified deliberately.
- Fit: AI-team docs already include decisions and implementation plans.
- Technical plan: add a homepage strip linking problem, decision, verification, and result docs.
- Estimated cost: low.
- Showcase effect: communicates judgment and QA.
- Risk: can feel internal if placed too prominently.

### 3. Role Readiness Scorecard

- User value: hiring visitors understand role fit quickly.
- Fit: the site already has proof and project sections.
- Technical plan: add rows for frontend, AI-assisted engineering, static publishing, and UX judgment, each with proof and next inspection link.
- Estimated cost: low.
- Showcase effect: direct recruiter value.
- Risk: self-scoring can feel inflated.

### 4. Interview Question Launchpad

- User value: interviewers get concrete questions to ask.
- Fit: existing case studies and code tour provide material.
- Technical plan: add prompt cards linked to relevant site sections.
- Estimated cost: low.
- Showcase effect: turns browsing into conversation.
- Risk: may feel artificial if copy is not natural.

### 5. Project Constraint Cards

- User value: technical reviewers see tradeoffs, not just outcomes.
- Fit: project cards already include evidence receipts.
- Technical plan: add constraint, tradeoff, and next improvement rows.
- Estimated cost: low.
- Showcase effect: shows mature thinking.
- Risk: can overcrowd project cards.

### 6. Curated Technical Archive Lanes

- User value: recruiters can find technical writing quickly.
- Fit: post data is already generated.
- Technical plan: add themed post lanes or recommended reads.
- Estimated cost: medium.
- Showcase effect: improves credibility through writing.
- Risk: post tags may be uneven.

### 7. Contact Intent Panel

- User value: lowers friction to contact.
- Fit: contact section already exists.
- Technical plan: add cards for recruiting, technical feedback, project discussion, and writing feedback.
- Estimated cost: low.
- Showcase effect: improves conversion.
- Risk: must avoid promising response times.

### 8. Account Identity Map

- User value: external accounts become easier to interpret.
- Fit: multiple account links already exist.
- Technical plan: regroup links by Code, Writing, Video, Notes, ACGN, and Contact.
- Estimated cost: low.
- Showcase effect: makes personal brand more coherent.
- Risk: not enough technical differentiation alone.

## Recommended Priority

1. Site Pipeline Evidence Lens
2. Decision Receipts Strip
3. Role Readiness Scorecard
4. Interview Question Launchpad
5. Project Constraint Cards
6. Contact Intent Panel
7. Curated Technical Archive Lanes
8. Account Identity Map
