# Round 05 Brainstorm - Agent 04 AI Applications Expert

Scope: AI should be represented honestly through workflow evidence and local UX, not through fake chatbot features or paid APIs.

## Direction Options

- Honest AI evidence layer: document where AI assisted decisions and where human review remained necessary.
- AI-inspired local navigation: make discovery clearer without external services.
- Process-as-product presentation: expose the site improvement pipeline.

Recommended combination: Site Pipeline Lens, AI Boundary Ledger, Search Match Explainer, and Curated Archive Lanes.

## Feature Ideas

### 1. Site Pipeline Lens

- User value: visitors understand the website as a local-first publishing system.
- Fit: existing editor/import/manifest/search/archive/guestbook workflow is real and inspectable.
- Technical plan: add homepage pipeline cards with links to the relevant pages, scripts, generated data, and docs.
- Estimated cost: medium.
- Showcase effect: shows pragmatic engineering and workflow design.
- Risk: too much internal process can reduce emotional appeal.

### 2. AI Boundary Ledger

- User value: visitors see responsible AI usage rather than vague "AI-powered" claims.
- Fit: AI-team docs already record decisions and implementation plans.
- Technical plan: add a small ledger with columns: AI role, human decision, runtime reality, evidence, limitation.
- Estimated cost: low.
- Showcase effect: strong personal brand signal for AI-assisted engineering.
- Risk: can feel defensive if copy is too formal.

### 3. Search Match Explainer

- User value: search becomes explainable.
- Fit: current local search can expose match reason without AI.
- Technical plan: label title, tag, excerpt, or body matches in result cards.
- Estimated cost: medium.
- Showcase effect: mimics explainable retrieval patterns.
- Risk: touches `script.js`.

### 4. Curated Archive Lanes

- User value: visitors get guided reading without an external recommender.
- Fit: posts already have metadata.
- Technical plan: add hand-authored lane definitions and render matching posts.
- Estimated cost: medium.
- Showcase effect: demonstrates product curation.
- Risk: manual lanes require maintenance.

### 5. Evidence Query Cards

- User value: visitors can click predefined queries like "technical notes" or "site workflow".
- Fit: local search already exists.
- Technical plan: add cards that populate the search box or jump to relevant sections.
- Estimated cost: low to medium.
- Showcase effect: improves discovery.
- Risk: requires JS if cards manipulate search.

### 6. Capability-to-Proof Matrix

- User value: recruiters connect capabilities to proof quickly.
- Fit: proof map and project receipts exist.
- Technical plan: add a compact table of capability, proof link, limitation, and next inspection step.
- Estimated cost: low.
- Showcase effect: strengthens credibility.
- Risk: may duplicate the Proof Map.

### 7. Local Recommended Reading Blocks

- User value: different visitor types get relevant posts.
- Fit: homepage already has routes for recruiter/reviewer/reader.
- Technical plan: add static reading cards under each route.
- Estimated cost: low.
- Showcase effect: improves content conversion.
- Risk: manual updates needed.

### 8. Project Assumption Cards

- User value: visitors see tradeoffs and constraints behind decisions.
- Fit: existing project cards have evidence rows.
- Technical plan: add native `details` blocks for assumptions and limitations.
- Estimated cost: low.
- Showcase effect: signals engineering maturity.
- Risk: can make the homepage too dense.

### 9. Account Identity Map

- User value: external platforms become more understandable.
- Fit: many account links already exist.
- Technical plan: regroup links by visitor intent.
- Estimated cost: low.
- Showcase effect: improves personal brand coherence.
- Risk: low technical value.

### 10. Hand-Written Brief Cards

- User value: visitors see concise personal notes instead of generic labels.
- Fit: the homepage has several proof sections that could benefit from stronger copy.
- Technical plan: add small editorial notes near key sections.
- Estimated cost: low.
- Showcase effect: more human and less dashboard-like.
- Risk: subjective tone.

## Recommendation

Best first feature: Site Pipeline Lens. Pair it with a minimal AI Boundary Ledger if space allows, because together they show both technical workflow and judgment.
