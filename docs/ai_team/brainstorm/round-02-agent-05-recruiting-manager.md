# Round 2 Brainstorm - Agent 5: Recruiting Manager

## 1. Hiring Snapshot

- User value: A recruiter immediately sees target roles, strongest evidence, and how to evaluate fit.
- Why it fits: The site has project proof but lacks a recruiter entry point.
- Technical implementation: Add a homepage section with role summary, skill pillars, location/availability if appropriate, and evidence links.
- Estimated cost: Low.
- Display effect: First-scroll clarity without becoming a resume clone.
- Risks: Overly broad wording can weaken credibility.

## 2. Capability Evidence Matrix

- User value: Hiring teams can connect claims to artifacts.
- Why it fits: Prevents skills from being unsupported labels.
- Technical implementation: Static matrix of capability, proof, depth, and link.
- Estimated cost: Low.
- Display effect: Strong credibility boost.
- Risks: Needs regular maintenance when projects change.

## 3. Project Proof Gallery

- User value: Reviewers get fast visual confidence in project seriousness.
- Why it fits: Case pages are text-heavy.
- Technical implementation: Add screenshot/diagram gallery with captions and local assets.
- Estimated cost: Medium.
- Display effect: Better skim value for busy hiring teams.
- Risks: Requires representative visuals.

## 4. Guided Code Review Route

- User value: Technical interviewers know where to look in each repo.
- Why it fits: External repos are linked, but the review path is not prominent enough.
- Technical implementation: Add short "review these files first" blocks with purpose and expected takeaway.
- Estimated cost: Low.
- Display effect: Makes the portfolio easier to verify.
- Risks: External repo drift.

## 5. Site as Project

- User value: Shows engineering process, not only outcomes.
- Why it fits: The site's AI-team docs and generation scripts are visible proof of process.
- Technical implementation: Add a static engineering notes page linked from footer or project list.
- Estimated cost: Low.
- Display effect: Portfolio itself becomes an artifact.
- Risks: Lower immediate value than hiring snapshot.

## 6. Ownership Receipts

- User value: Shows what the candidate personally owned.
- Why it fits: Personal sites often fail to distinguish ownership from participation.
- Technical implementation: Add "owned by me" bullets to case pages and matrix rows.
- Estimated cost: Low.
- Display effect: Higher trust for recruiters.
- Risks: Must stay factual.

## 7. Engineering Decision Log

- User value: Reveals judgment and tradeoff thinking.
- Why it fits: Case studies already discuss tradeoffs; a cross-site decision log makes this easier to scan.
- Technical implementation: Static list of decisions with context, options, choice, and link.
- Estimated cost: Low to medium.
- Display effect: Signals senior engineering behavior.
- Risks: Can read as self-important if verbose.

## 8. Recruiter FAQ

- User value: Answers common screening questions quickly.
- Why it fits: Helps non-technical visitors navigate.
- Technical implementation: Add compact FAQ details/summary section.
- Estimated cost: Low.
- Display effect: Reduces friction.
- Risks: Too much personal/process detail could distract.

## 9. Learning Trajectory

- User value: Shows growth direction and consistency.
- Why it fits: Existing posts can support a timeline.
- Technical implementation: Static timeline or generated topic chronology.
- Estimated cost: Medium.
- Display effect: Makes the site feel alive.
- Risks: Depends on content density.

## Priority Recommendation

Implement Hiring Snapshot + Capability Evidence Matrix first. It is the most direct conversion improvement after adding case studies.
